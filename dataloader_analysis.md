# dataloader.py 详细分析文档

## 文件概述

`dataloader.py` 是 GNN-UDS 项目的核心数据加载与预处理模块，主要用于处理城市排水系统模拟数据，为深度学习模型（特别是GNN模型）准备训练和测试数据。该模块负责：

1. **数据生成**：通过 SWMM 模拟生成排水系统状态数据
2. **数据加载**：从文件加载预生成的模拟数据
3. **数据预处理**：对数据进行序列化、批处理、归一化等操作
4. **数据管理**：数据的存储、更新和清理

## 类结构：DataGenerator

### 主要属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `config` | dict | 环境配置参数 |
| `data_dir` | str | 数据存储目录 |

| `ctrl_step` | int | 控制步长 |
| `limit` | int | 数据容量限制 |
| `cur_capa` | int | 当前数据容量 |

## 核心方法详解

### 1. `__init__` - 初始化方法
初始化数据生成器，设置各种配置参数和路径。

**参数：**
- `env_config`: 环境配置字典
- `data_dir`: 数据存储目录
- `args`: 额外的参数对象

### 2. `simulate` - 模拟方法
执行单次SWMM模拟，生成排水系统状态数据。

**参数：**
- `env`: 环境对象
- `event`: 暴雨事件
- `seq`: 是否为序列模式
- `act`: 是否包含控制动作
- `hotstart`: 热启动时间（分钟）

**返回：**
- `states`: 系统状态序列
- `perfs`: 性能指标序列（洪水情况）
- `settings`: 控制设置序列（如果act=True）
- `rains`: 降雨序列
- `edge_states`: 边状态序列

### 3. `generate` - 批量生成方法
批量生成多个暴雨事件的数据，支持多进程并行。

**参数：**
- `events`: 暴雨事件列表
- `processes`: 进程数
- `repeats`: 每个事件的重复次数
- `seq`: 是否为序列模式
- `act`: 是否包含控制动作

### 4. `prepare_batch` - 准备批次数据
为训练准备批次数据，支持序列输入输出。

**参数：**
- `event_idxs`: 事件索引
- `seq`: 序列长度
- `batch_size`: 批次大小
- `interval`: 采样间隔
- `continuous`: 是否连续采样
- `trim`: 是否修剪序列
- `return_idx`: 是否返回索引

**返回：** 包含各种数据项的列表

### 5. `state_split_batch` - 状态切分方法
将状态数据切分为输入、边界条件和输出三部分。

**参数：**
- `states`: 状态元组（前一时刻，当前时刻）
- `perfs`: 性能指标元组
- `trim`: 是否修剪序列

**返回：**
- `X`: 输入数据（水位、上游流量、下游流量、降雨）
- `B`: 边界条件（降雨、潮位）
- `Y`: 输出数据（下一时刻状态+性能指标）

### 6. `get_norm` - 归一化参数计算
计算数据的归一化参数（最大值、最小值）。

**返回：** 包含各数据项归一化参数的列表

## 数据处理流程

### 1. 数据生成流程
```
事件列表 → 多进程SWMM模拟 → 原始数据收集 → 后处理 → 存储为.npy文件
```

### 2. 训练数据准备流程
```
加载.npy文件 → 计算归一化参数 → 生成批次索引 → 准备批次数据 → 返回给模型
```

## 数据格式说明

### 1. 系统状态数据 (`states`)
- 形状: `[时间步数, 节点数, 4]`
- 4个维度: 水位(h)、总入流(q_totin)、下游流量(q_ds)、降雨(r)

### 2. 性能指标 (`perfs`)
- 形状: `[时间步数, 节点数]`
- 表示: 洪水深度（>0表示洪水）

### 3. 降雨数据 (`rains`)
- 形状: `[时间步数, 降雨站数]`

### 4. 边状态数据 (`edge_states`)
- 形状: `[时间步数, 边数, 2]`
- 2个维度: 流量、水深

## 关键特点

### 1. 序列数据处理
支持长短时记忆（LSTM）风格的序列输入输出：
- 输入序列长度: `seq_in`
- 输出序列长度: `seq_out`
- 使用`expand_seq`方法进行序列扩展

### 2. 不平衡数据处理
洪水事件通常是不平衡的：
- 使用`get_flood_weight`计算样本权重
- 洪水样本权重更高，确保模型学习洪水情况

### 3. 多进程支持
- 使用`multiprocessing.Pool`并行模拟
- 加速数据生成过程

### 4. 增量学习支持
- 使用`update`方法添加新数据
- 自动管理数据容量限制
- 支持测试集隔离

## 与main.py的关系

在`main.py`中，`DataGenerator`被用于：
1. **训练阶段**: 生成或加载训练数据
2. **测试阶段**: 加载测试数据
3. **增量学习**: 在线更新数据

## 使用示例

```python
# 初始化数据生成器
data_gen = DataGenerator(
    env_config=config,
    data_dir='./data/',
    args=args
)

# 生成数据
data_gen.generate(events, processes=4, repeats=1, seq=True, act=True)

# 准备训练批次
event_idxs = data_gen.get_data_idxs(seq=5)
batch = data_gen.prepare_batch(event_idxs, seq=5, batch_size=32)

# 保存数据
data_gen.save()
```

## 注意事项

1. **内存管理**: 设置了`limit`参数限制数据量，防止内存溢出
2. **文件存储**: 使用`.npy`格式存储，支持内存映射读取
3. **随机采样**: 支持加权随机采样，处理数据不平衡问题
4. **序列对齐**: 确保输入输出序列的时间对齐
